# Machines and Power

Excessive Utilities uses both **Grid Power** and **FE**. They solve different problems.

| Power type | Used for | Stored? |
| --- | --- | --- |
| Grid Power | Player-owned utility effects and machine reservation costs. | No. It is a live capacity budget. |
| FE | Conventional energy machines, batteries, generators, and wireless FE. | Yes. It can be stored and transferred. |

## Grid Power Machines

GP-powered machines reserve GP while they have work to do. If the grid is overloaded, machines that check overload pause until enough GP is available.

The **Resonator** is the most important early GP machine. It uses the GP cost from the active recipe, plus any Speed Upgrade cost. One Speed Upgrade stack makes it craft faster, but also increases the GP required to keep it running.

## FE Machines

FE generators and FE machines use conventional energy. Generator blocks are owner-aware for some effects and networks, but they do not add Grid Power.

Wireless FE is still tied to ownership:

- **Wireless FE Batteries** store energy for the owner's wireless network.
- **Wireless FE Transmitters** pull from that network and send FE to nearby receivers.
- Each transmitter destination that accepts FE costs GP while it is being powered.

## Speed Upgrades

Speed Upgrades increase machine work per tick. By default, their GP cost scales with stack size:

```text
stackSize * (122 + 4 * stackSize) / 126.0
```

The important player-facing rule is simple: more speed needs more spare GP. If a machine works without upgrades but pauses after upgrades, add GP generation or reduce the upgrade stack.

## Common Checks

If a machine is not working:

- Check that the input has a valid recipe.
- Check that the output slot can accept the result.
- Check your `used / capacity GP` display.
- Use the Power Manager to see the exact active GP drains.
- For FE machines, check stored energy and transfer direction.

# KubeJS and Config

Most server-facing balance values are controlled by the server config. Use config first when you only need to change numbers such as GP generation, GP usage, ring timings, generator output, quarry rates, or Soul Fragment health.

KubeJS is only needed for behavior that the config does not directly expose.

## Speed Upgrade GP Formula

Speed Upgrades have a configurable multiplier in the server config:

```text
grid_power.gp_usage.speedUpgradeGpCostMultiplier
```

The formula itself can also be replaced from KubeJS by calling `SpeedUpgradeItem.setGpCostCalculator`.

```js
const $SpeedUpgradeItem = Java.loadClass('dev.aaronhowser.mods.excessive_utilities.item.SpeedUpgradeItem')

$SpeedUpgradeItem.setGpCostCalculator(stackSize => stackSize * 10)
```

That example makes each Speed Upgrade in the stack cost 10 GP before the server config multiplier is applied.

## Useful Config Areas

| Config section | Controls |
| --- | --- |
| `general.soul_fragments` | Soul Fragment max-health value and death reset behavior. |
| `grid_power.gp_generation` | GP produced by mills and panels. |
| `grid_power.gp_usage` | GP used by rings, Red Coal, Speed Upgrades, and Wireless FE. |
| `rings` | Chicken Wing Ring and Ring of the Flying Squid timings and movement values. |
| `machines` | FE machines, quarries, QED, wireless FE, and furnace settings. |
